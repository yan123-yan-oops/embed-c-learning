## 总体策略

### 为什么 C 和 MATLAB 要并行？

> 嵌入式采集数据（C代码） + 上位机验证算法（MATLAB）= 完整闭环

实际工作流：
```
北斗卫星 ─→ RF前端 ─→ ADC ─→ FPGA/ARM(C代码采集) ─→ PC/MATLAB(算法验证)
                              ↑ 你的工作范围                        ↑ 你的仿真
```

**并行优势**：
- C 学的每个概念，MATLAB 里能"看到"——指针就是数据地址，缓冲区就是波形窗口
- MATLAB 学的每个算法，C 里知道它跑在什么硬件上、有多少算力限制
- 第 4 个月做抗干扰仿真时，你已经有 3 个月的 MATLAB 手感了

## 阶段 1：C 筑基 + MATLAB 入门（第 1 月）

### 每周分配

| 周 | C / 嵌入式（~70% 时间） | MATLAB / 信号（~30% 时间） |
|----|----------------------|--------------------------|
| **W1** | 01-basics 基本类型 / volatile / static | 安装 MATLAB → 矩阵运算、plot 画图 |
| **W2** | 02-pointers 指针 / 函数指针 | 04-memory 内存布局 | 生成正弦波、读取 .bin 文件 |
| **W3** | 03-bitwise 位操作 / 寄存器宏 | 05-struct-enum 结构体 / union / packed | 频谱分析 fft / 加噪声 |
| **W4** | 06-strings NMEA 字符串解析 | 07-ringbuffer 环形缓冲区 | 09-pool-allocator 静态内存池 |

#### 面试回答框架（STAR）

```
S（场景）：在北斗导航采集项目中，我们需要从 UART 接收 GPS 模块数据
T（任务）：数据流要确保不丢、不断，中断和主循环协同一致
A（行动）：我设计了环形缓冲区，中断只管写，主循环只管读，无锁安全
          上层用静态内存池管理消息块，彻底不用 malloc
R（结果）：系统连续运行 72 小时无丢包、无内存泄漏
          后来迁移到 FreeRTOS，只把 rb_put/rb_get 换成 xQueueSend/xQueueReceive
```
